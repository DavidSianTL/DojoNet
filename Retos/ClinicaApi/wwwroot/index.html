🚀 API REST - Clínica Médica

Objetivo General: Implementar y consumir una API REST que gestione información y operaciones de una clínica, incluyendo pacientes, médicos, citas y especialidades.

📘 Recursos y Relaciones

Recurso Campos Principales Relaciones
Pacientes id, nombre, email, telefono, fechaNacimiento Un paciente puede tener muchas citas
Médicos id, nombre, especialidad_id, email Un médico tiene muchas citas y pertenece a una especialidad
Especialidades id, nombre Una especialidad puede tener muchos médicos
Citas id, paciente_id, medico_id, fecha, hora Relación entre paciente y médico

🔗 Endpoints principales (CRUD)

Operación Endpoint Método Descripción
Listar pacientes /api/pacientes GET Lista todos los pacientes
Crear médico /api/medicos POST Crea un nuevo médico
Modificar cita /api/citas/{id} PUT Actualiza datos de una cita
Eliminar especialidad /api/especialidades/{id} DELETE Elimina una especialidad
Detalle paciente /api/pacientes/{id} GET Obtiene detalle individual

🛠️ Implementación de la API

  • ✅ Modelos en .NET Framework
  • ✅ Relaciones mediante claves foráneas
  • ✅ Controladores con validación y manejo de errores

🔐 Seguridad básica

  • JWT o token simple para proteger endpoints
  • POST /api/login para autenticación
